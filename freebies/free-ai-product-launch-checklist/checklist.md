## PHASE 1: AI Product Idea Worksheet

*Fill this in before building anything. If you can't answer these cleanly, your product isn't ready yet.*

**Step 1: Write your product in one sentence**

> My product helps _______________________ (who)
> do _______________________ (what specific thing)
> so they can _______________________ (what result they care about)

Example: *"My product helps data science students build a complete fraud detection system so they can add a real portfolio project to their GitHub in a weekend."*

**Step 2: What is the specific pain your buyer has right now?**

Write 3 frustrations your buyer has before they find your product:
1. _______________________________________________
2. _______________________________________________
3. _______________________________________________

**Step 3: What does the buyer get? (be specific)**

List every file, document, template, notebook, or asset included:
- [ ] File 1: _______________________________________________
- [ ] File 2: _______________________________________________
- [ ] File 3: _______________________________________________

**Step 4: What is the format?**
- [ ] PDF guide
- [ ] ZIP with code files
- [ ] Jupyter Notebook
- [ ] Template files
- [ ] Prompt collection
- [ ] Other: _______________

**Step 5: What can the buyer accomplish in 10 minutes after downloading?**

> In 10 minutes, my buyer will: _______________________________

*If you can't answer this, your onboarding is not clear enough yet.*

## PHASE 4: Product Page Quality Check

Read your product page as if you are the buyer, seeing it for the first time.

**The 5-second test:** After 5 seconds on the page, can the buyer answer:
- [ ] What is this? (clear from the headline)
- [ ] Who is it for? (clear from subheadline or first paragraph)
- [ ] What do I get? (clear from the "what you get" section)
- [ ] How will it help me? (clear from the problem section)
- [ ] Why should I trust this? (clear from the trust section)

**The honesty check:**
- [ ] No fake testimonials
- [ ] No fake sales counts ("Over 1,000 students!")
- [ ] No income claims ("Make $5,000 this month!")
- [ ] No manufactured urgency ("Only 3 left!") for a digital product
- [ ] No placeholder text (TODO, INSERT HERE, [BRACKETED])

**If any of the above fail — fix them before publishing.**

## PHASE 6: First GitHub README Checklist

*If your product includes code, your GitHub README is part of your credibility.*

- [ ] Repository name is clear and professional
- [ ] README starts with what the project does (not installation instructions)
- [ ] README includes a screenshot or output example (even a text-based one)
- [ ] Prerequisites listed: Python version, libraries, OS requirements
- [ ] Installation instructions are copy-pasteable (exact commands, no placeholders)
- [ ] Usage section shows a real example (not `python example.py`)
- [ ] License included (MIT is standard for open resources)
- [ ] Contact section: your name and email or LinkedIn

| 7 | If no sales: check your product page against Phase 4 checklist. What's the weakest section? Fix it. | [ ] |

**If you have no page views after 7 days:** The problem is distribution, not the product. You need more people to see it. Post in one more community, share with one more person, or add the product link to your GitHub profile bio.

**If you have page views but no sales:** The problem is the product page. Go back to Phase 4 and find the section that isn't convincing enough.
